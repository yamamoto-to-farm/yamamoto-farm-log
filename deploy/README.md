1. 播種・育苗
初期工程。作業内容が明確で、入力項目も独立しているため2ページ構成。

播種ログ：播種日、品種、数量、育苗ハウスなど

育苗ログ：育苗管理、移動、潅水、温度管理など

2. 圃場管理（畑を作る・整える）
トラクター作業や圃場整備など、畑そのものを整える作業をまとめたカテゴリ。

● 土づくり・耕起ログ
耕うん・サブソイラーなど、土を動かす作業を統合。
入力項目がほぼ共通（圃場・深さ・速度・機械）。

● 除草・草刈りログ
草刈りと除草剤散布を統合。
パートさんが入力しやすい構造。

● 圃場整備ログ
法面補修・側溝土上げ・明渠掘り・伐採など、
整備作業を1ページに集約。

3. 栽培管理（作物を育てる）
作物の生育に直接関わる作業をまとめたカテゴリ。

● 施肥ログ
改良材・元肥・追肥・緑肥・堆肥・フロンサイドなど
肥料投入に関わる作業をすべて統合。

● 防除ログ
殺虫剤・殺菌剤・液肥・BT資材など、
ブームスプレーヤで散布する作業を統合。

● 潅水ログ
潅水時間・量など、軽い入力で済む作業。

● 除草ログ（手作業）
畝間の草取りなど、手作業の除草。

● 中耕ログ（除草・追肥オプション）
中耕を主作業とし、

中耕のみ

中耕＋除草

中耕＋追肥
をオプションで吸収。

● 畝立てログ（元肥オプション）
畝立てを主作業とし、

畝立てのみ

畝立て＋元肥
をオプションで吸収。

4. 定植
植え付け作業。
作業内容が明確で独立しているため1ページ構成。

5. 収穫・出荷
収穫と出荷は作業内容が大きく異なるため、2ページ構成。

🎯 この構造のメリット
ページ数が増えすぎない（約12ページに集約）

作業者が迷わない（動線が明確）

入力項目の共通性でまとめている

データ構造がシンプル

後継者が理解しやすい
