# 年間作付計画ページ（3ステップ構成）ロードマップ  
Yamamoto Farm OS – Annual Crop Planning Module  
作成日: 2026-05-01  
作成者: 直輝

---

## ■ 概要

本モジュールは、  
**5月の大枠契約 → 種の注文 → 7月以降の播種計画**  
までを一貫して支援するための “年間作付計画レイヤー” を OS に追加するもの。

これにより、紙で行っていた以下の作業を完全にデジタル化する。

- 年間の出荷計画  
- 月別・週別の収穫計画  
- 品種ごとの作付反数  
- 必要播種株数・トレイ枚数  
- 育苗ハウスの概算キャパ  
- 播種計画への落とし込み  

---

# STEP1：月別の収穫目標 → 作付予定面積の算出  
**目的：5月の大枠契約に必要な「年間の作付規模」を決める**

### ▼ 入力（ユーザー）
- 月別の目標出荷基数（鉄コン基数）  
  - 11月〜翌4月（寒玉・レッド）  
  - 翌5月〜翌6月（初夏キャベツ）
- 月別の 1反あたり目標基数  
- 月別の目標反収（kg/10a）  
- 品種割合（任意）

### ▼ 計算（OS）
- 月別の必要反数  
- 品種別の必要反数（割合使用時）  
- 年間の総反数  
- 年間の総出荷量（t）  
- 契約数量とのギャップ表示

### ▼ 出力
- 「年間作付フレーム（収穫月ベース）」  
  - 例：  
    - 11月：10反  
    - 12月：15反  
    - 1月：12反  
    - 2月：8反  

---

# STEP2：週別に品種 × 基数を割り当てる（ロット設計）  
**目的：収穫バランス・品種配置・生育逆算を行う**

### ▼ 入力（ユーザー）
- 各週に収穫したい品種  
- 各週の目標基数  
- 品種データ（varieties-detail.json）

### ▼ 品種データ構造（新規）
`varieties-detail.json`
    {
        "藍天": {
            "2025-plantingRef-01": {
                "sowDate": "",
                "plantDate": "",
                "harvestStart": "",
                "harvestEnd": "",
                "harvestDays": 45,
                "growDays": 120,
                "yieldPer10a": 6500,
                "field": "A-3"
            }
        },
        "冬藍": { ... }
    }


### ▼ 計算（OS）
- 品種ごとの必要反数（週単位）  
- 収穫週 → 定植日 → 播種日の逆算  
- 必要株数  
- 必要トレイ枚数（128/200）  
- 週別の播種負荷  
- 週別の育苗ハウス占有率（概算）  
- 週別の定植負荷（面積）

### ▼ 出力
- 「週別ロット計画」  
- 「品種別の定植予定面積」  
- 「品種別の播種株数・トレイ枚数」  
- 「育苗ハウス負荷グラフ（概算）」  

---

# STEP3：播種計画・定植計画への落とし込み  
**目的：年間フレーム → seedList（播種計画）へ自動変換**

### ▼ OS が生成するもの
- planSowDate（逆算）  
- variety  
- trayCount  
- trayType  
- planArea  
- daysToPlant  
- planPlantDate  
- harvestPlanYM  

### ▼ チェック（OS）
- 必要トレイ枚数  
- 育苗ハウスの回転  
- 定植面積  
- 圃場候補の提示（自動決定はしない）

### ▼ ユーザーが決めるもの
- 圃場の最終決定（自動化しない）  
- 播種日の微調整  
- 品種の最終調整  

### ▼ 出力
- seedList/all.csv の自動生成  
- 定植計画（plantingList）への連携  
- 年間作付計画との整合性チェック  

---

# データ構造まとめ

## varieties-detail.json（新規）
- 品種 × 過去の plantingRef の入れ子構造  
- 生育日数・反収・収穫期間・圃場履歴を保持  
- 年間作付計画の基礎データとして使用

## annual-plan.json（新規）
- STEP1〜STEP3 の中間データを保持  
- seedList 生成前の「年間フレーム」を保存

# ロードマップ（実装順）

## Phase 1：基盤整備
- varieties-detail.json の設計・生成  
- annual-plan.json の設計  
- 年間作付計画ページの UI 骨格（3カード）

## Phase 2：STEP1 実装
- 月別目標基数入力  
- 反収・1反基数入力  
- 必要反数の自動計算  
- 契約数量とのギャップ表示

## Phase 3：STEP2 実装
- 週別ロット割当 UI  
- 品種データ参照  
- 生育逆算（収穫→定植→播種）  
- 必要株数・トレイ枚数計算  
- 育苗ハウス負荷の概算チェック

## Phase 4：STEP3 実装
- seedList への自動生成  
- 定植計画への連携  
- 圃場候補の提示  
- 年間フレームとの整合性チェック

## Phase 5：運用改善
- 年間計画と実績の比較  
- 品種データの自動更新（反収・生育日数）  
- 翌年計画へのフィードバック
