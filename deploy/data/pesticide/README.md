# pesticide data README

このフォルダは農薬マスタを管理します。

## 対象ファイル
- `pesticide-index.json`
  - 一覧用の基本情報（ID / 名称 / カテゴリ / 単位）
- `pesticide-detail.json`
  - 詳細情報（登録番号、剤型、価格履歴、希釈倍率、使用制限など）

## ID ルール
ID はカテゴリ別プレフィックス + 4桁連番を使います。

形式:
- `CCNNNN`
- 例: `FG0001`

プレフィックス:
- `FG`: 殺菌剤
- `IN`: 殺虫剤
- `HL`: 茎葉除草剤
- `SL`: 選択制除草剤
- `AD`: 展着剤
- `SD`: 土壌消毒剤

運用ルール:
- 新規登録時はカテゴリに対応するプレフィックスで採番する
- 連番はカテゴリごとに増やす（例: FG0001, FG0002 ...）
- 一度採番した ID は固定し、後からカテゴリが変わっても原則変更しない

## カテゴリ ルール
カテゴリは次の6種類に統一します。
- 殺菌剤
- 殺虫剤
- 茎葉除草剤
- 選択制除草剤
- 展着剤
- 土壌消毒剤

入力時の注意:
- カテゴリ表記は完全一致で入力する
- 追加カテゴリが必要な場合は、先にこの README とプレフィックス表を更新する

## データ整合性チェック
保存前に以下を確認します。
- `pesticide-index.json` の `id` と `pesticide-detail.json` のキーが対応している
- `pesticide-index.json` の `category` がカテゴリ6種のいずれか
- `id` が `CCNNNN` 形式に一致している
